## 搬瓦工 64MB 内存 + WordPress：为什么"能跑"但不能用

WordPress 本身对内存的要求并不夸张。程序本体跑起来只占 8-16MB，安装阶段至少需要 32MB。所以从纯技术层面讲，64MB 的 VPS 理论上是可以装 WordPress 的——前提是用 lighttpd 替代 Apache 或 Nginx，把 MySQL 配置改到极限，php-cgi 进程数压到 2-3 个，内存 limit 调到最低。

但"跑起来"和"能用"是两回事。

64MB 里，操作系统本身就要吃掉 30-40MB，MySQL 再占一块，剩下的内存根本不够 WordPress 正常处理请求。稍微来几个并发访问，就开始 swap，然后卡，然后 500。更别说现在的 WordPress 版本对运行环境要求高多了：推荐 PHP 8.3+，推荐 MySQL 8.0+ 或 MariaDB 10.6+。而 MySQL 5.6 以上的版本，装起来就需要 1GB 内存起步，不然安装都能失败。

这一段说的是事实，不是吓你，而是帮你省掉一个走弯路的决策。

## 搬瓦工 WordPress 建站需要注意什么？

先说 KVM 入门套餐的实际使用情况。1GB 内存跑 WordPress，装个宝塔面板配置 LNMP 环境，日常运作是没问题的。宝塔的一键建站功能对新手友好，不用手动配置 Nginx 和 MySQL，点几下就搞定。

不过有几点坑要提前说：

**① 缓存插件是必装的。**
WordPress 每次访问都要跑一遍 PHP + 数据库查询，没有缓存的 TTFB（首字节时间）普遍 800ms 起步，开了缓存之后能降到 200-400ms，体感差距很大。WP Super Cache 免费够用，WP Rocket 付费但效果更好，可以根据预算选。

**② MySQL 配置要调。**
1GB 内存的 VPS 跑 MySQL 5.7 或 MariaDB，默认配置比较激进。建议把 `innodb_buffer_pool_size` 调小，不然偶尔会 OOM，WordPress 后台操作卡住。装完宝塔之后去看看内存监控，确认 MySQL 占用稳定才算到位。

**③ Swap 建议开。**
KVM 套餐默认没有 Swap，1GB 内存偶尔还是会有压力。建一个 1-2GB 的 Swap 文件能有效缓解内存不足时的崩溃问题。速度是慢一点，但比直接 OOM 强多了。

**④ PHP 内存限制要改。**
WordPress 默认 PHP memory_limit 是 64MB，单站点 WP_MEMORY_LIMIT 是 40MB。装了主题和插件之后，这个数字经常不够用，建议改到 128MB 或 256MB，在 `wp-config.php` 里加一行 `define('WP_MEMORY_LIMIT', '256M')` 就行。

## 关于"64M"这个疑问的补充说明

很多搜索"搬瓦工 64M WordPress"的人，其实混淆了两个"64M"：

一种是**VPS 的物理内存 64MB**——这是搬瓦工早年绝版套餐的配置，现在已经买不到了。

另一种是 **WordPress 的 PHP memory_limit 默认值 64MB**——这个是 WordPress 程序本身在 `wp-config.php` 里设置的 PHP 内存上限，跟你买的 VPS 内存大小是两回事。很多人在后台看到"建议增加 PHP 内存限制"的提示，这说的是这个，改一行配置就能解决，不需要升级套餐。

搞清楚这两个区别，能省不少折腾时间。
